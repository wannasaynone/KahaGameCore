# GameFlowSystem 新專案實作指南

按本文逐步操作，可在全新 Unity 專案重現 ProjectII 等級的完整內容：表驅動流程、七張表、整套 UGUI 介面（含 View/Presenter）、對話系統串接。本文自包含（程式碼全文皆收錄），搭配 `README.md`（架構與 API 總覽）與 `DialogueSystem/README.md` 閱讀。

## 步驟總覽

- [ ] **0.** 複製 KahaGameCore 包、安裝 Unity 套件、調整專案設定
- [ ] **1.** 建立遊戲程式碼 asmdef
- [ ] **2.** 準備七張表（JSON）
- [ ] **3.** 實作 View 層（9 個腳本）＊路線 A 可跳過
- [ ] **4.** 實作 Presenter 層（4 個轉接 + 1 個演出範例）＊路線 A 可跳過
- [ ] **5.** 實作組裝根 GameLauncher ＊路線 A 可跳過
- [ ] **6.** 建立 Prefabs 與場景 ＊路線 A 改為一鍵生成
- [ ] **7.** 啟動驗收

## 路線選擇

**路線 A：一鍵生成（推薦起步）** —— 使用包內 `DefaultViews/` 模組（預設 View / Presenter / `DefaultGameLauncher` 腳本 + UI builder）：

1. 完成第 0 節（包與設定）。第 1 節的 asmdef 在你開始寫自己的程式碼前可以省略。
2. 執行選單 **KahaGameCore → GameFlowSystem → Build Default UI Prefabs And Scene**，自動產生：
   - `Assets/Resources/GameFlowUIViews/` 下 9 個 View prefab（引用包內 DefaultViews 腳本）
   - `Assets/Scenes/GameFlowGame.unity`——Camera、EventSystem、4K Canvas、DialogueView、`DefaultGameLauncher` 全部接好，**並已把包內 `DefaultViews/SampleData/` 七張測試表拖進 `gameDataTables` 欄位**
3. 開啟該場景直接 **Play**——測試表是一個可完整遊玩的迷你循環（照顧小屋一週：料理/打掃/打工/採集、第二天解鎖外出、撐到第八天看製作名單；也涵蓋選項對話、GameEffect、Monologue、ShowHint、Game Over 等所有系統功能，可當表格寫法範例）。
4. 製作自己的內容：照第 2 節規格寫七張表，替換 DefaultGameLauncher 欄位裡的 TextAsset（留空則 fallback 到 `Resources/GameData/{型別名}.txt`）。

之後的客製：
- 版面美術：直接改生成的 prefab（**重跑 builder 會覆寫**，改完別重跑、或先備份）。
- 組裝邏輯（Override 服務、註冊演出、自訂指令）：把 `DefaultViews/DefaultGameLauncher.cs` 複製到專案、改名改命名空間後修改，替換場景中的組件——不要直接改包內版本。
- UI 行為：同樣把對應 View/Presenter 複製出去改，prefab 上的組件換成你的版本。

**路線 B：完全自有 UI** —— 不用 DefaultViews，照第 3~6 節手動實作所有 View / Presenter / 組裝根。以下章節的程式碼即 DefaultViews 的原始碼（namespace 不同而已），也可以直接抄包內檔案。

## 0. 前置：包、套件與專案設定

### 0.1 複製 KahaGameCore

整個 `Assets/KahaGameCore/` 複製到新專案。內含模組與 asmdef 名稱（asmdef reference 要用「內部名稱」，不是檔名）：

| 模組 | asmdef 內部名稱 | 用途 |
|---|---|---|
| `Package/GameFlowSystem/Scripts` | `KahaGameCore.Package.GameFlowSystem` | 流程核心（GameFlowController + 介面） |
| `Package/GameFlowSystem/DefaultImplements` | `KahaGameCore.Package.GameFlowSystem.DefaultImplements` | 表驅動預設實作 + Builder |
| `Package/EffectProcessor` | `KahaGameCore.Package.EffectProcessor.EffectProcessor` / `.Data` / `.Processor` | 效果指令引擎 |
| `Package/UserInterfaceSystem` | `KahaGameCore.Package.UserInterfaceSystem` | AView / UserInterfaceController |
| `Package/DialogueSystem` | `ProjectBSR.DialogueSystem`（歷史名稱） | 對話系統 |
| `InGame/GameData` | `KahaGameCore.GameData` | GameStaticDataManager / IGameData |
| `InGame/ValueContainer` | ⚠️ `KahaGameCore.Package.ValueContainer`（檔名是 `KahaGameCore.ValueContainer.asmdef`，內部名稱不同！） | 數值容器 + Calculator |
| `GameEvent` | `KahaGameCore.GameEvent` | EventBus |
| `Plugins/UniTask` | `UniTask` | 隨包附帶，**毋需另外安裝** |

### 0.2 Unity 套件（Package Manager）

| 套件 | 必要性 |
|---|---|
| Input System (`com.unity.inputsystem`) | 必要（EventSystem 使用 `InputSystemUIInputModule`） |
| Addressables (`com.unity.addressables`) | 必要（DialogueSystem 預設 CG/音訊 Provider 依賴；即使暫不用 CG 也要裝，否則 DialogueSystem 編不過） |
| TextMeshPro | Unity 6 內建（UGUI 2.0） |

### 0.3 ProjectSettings

- **Active Input Handling = Both**（Edit → Project Settings → Player → Other Settings）。DialogueView 的 `Update()` 用舊版 `UnityEngine.Input`，新版 UI 模組又需要 Input System。**改完必須重啟編輯器**。
- 中文顯示：自建中文 TMP Font Asset（Window → TextMeshPro → Font Asset Creator），否則 TMP 預設字型下中文全是方塊。

## 2. 七張表

每張表是一個 **JSON 陣列** 的 .txt（每列一個物件，欄位名稱區分大小寫）。對應 Google Sheet 分頁匯出。

### 2.1 載入方式（二擇一）

- **手動指定（推薦）**：放任意資料夾，GameLauncher 以 `TextAsset[]` 欄位拖入，用 `TextAssetJsonStaticDataHandler` 載入。**檔名必須與資料型別名稱一致**（`TimePhaseData.txt` ↔ `TimePhaseData`）。
- **Resources 路徑**：放 `Resources/GameData/{型別名}.txt`，用 `ResourcesJsonStaticDataHandler`。

### 2.2 共通語法

**條件式**（Condition / VisibleCondition / EnableCondition 欄位）：

- `$標籤` 引用 GameValueData 定義的數值：`$Supplies >= 200`
- 比較：`>=` `<=` `==` `!=` `>` `<`；邏輯：`&&` `||`（`&&` 優先，不支援括號）
- 兩側可寫四則運算與 `Random(min,max)`：`$Supplies+$TradeCount*10 > 100`
- **留空白 = 恆成立**
- 系統保留標籤：`$Day`、`$CurrentPhase`（時段ID）、`$CurrentLocation`（地點ID）、`$EventDone_{事件ID}`、`$LocationUnlocked_{地點ID}`

**效果指令串**（Commands 欄位）：多指令以 `;` 分隔；**系統會剝除所有半形空格**，參數內不可放帶空格的文字（文字一律放 GameTextData 以 ID 引用）；數值參數可用 `$標籤` 與四則運算。

| 指令 | 說明 |
|---|---|
| `AddValue(標籤,公式)` | 數值加上公式結果（可為負），自動鉗制上下限 |
| `SetValue(標籤,公式)` | 數值設為公式結果 |
| `AdvanceTime()` | 推進到下一時段（依 TimePhaseData.NextID，必要時換日） |
| `SetPhase(階段Key)` | 直接跳到指定階段（不換日） |
| `MoveToLocation(地點ID)` | 移動地點（流程會補發 EnterLocation 事件） |
| `StartDialogue(對話ID)` | 播放劇情對話並等待結束 |
| `ShowHint(文字ID)` | 提示視窗（文字取自 GameTextData），等待玩家確認 |
| `Monologue(群組名)` | 從 GameTextData 依群組+條件加權隨機抽一句顯示於 HUD |
| `PlayPerformance(演出ID)` | 播放 UGUI 演出（見第 7 節） |
| `OpenLocationMenu()` | 開啟移動選單讓玩家選地點（可取消） |
| `ReturnToTitle()` | 中止流程返回主標題（結局用） |
| `Wait(秒數)` | 暫停指令串指定秒數 |

新增指令：實作 `EffectCommandBase` → 以 `GameFlowSystemBuilder.AddCommandRegistration()` 註冊。

**觸發時機**（GameEventTriggerData.Timing）：`GameStart`、`PhaseStart:{階段Key}`、`AfterAction:{行動ID}`、`EnterLocation:{地點ID}`。冒號後寫保留字 `Any` 可命中同類別所有時機（如 `PhaseStart:Any` 做 Game Over 檢查，配高 Priority）——因此階段 Key 與行動 ID 不可命名為 `Any`。

### 2.3 表格欄位規格

**TimePhaseData（時間階段）**：`ID`、`Key`（英文識別字，`PhaseStart:{Key}`/`SetPhase(Key)` 引用）、`DisplayName`、`NextID`（推進後的階段）、`IsNewDay`（1=換日）、`AllowAction`（1=開放行動；0=事件後自動推進）、`Note`

**GameValueData（數值定義）**：`ID`、`Tag`（`$Tag` 引用）、`DisplayName`、`InitialValue`、`MinValue`/`MaxValue`（自動鉗制）、`ShowInHUD`（1=顯示於狀態列）、`Note`。新增旗標/數值直接加列，程式不用改。

> 時段自然消耗（每時段扣飽腹/精神）不再是內建欄位，改由資料表事件實現：在 GameEventTriggerData 加一筆 `Timing=PhaseStart:Any`、`Commands=AddValue(Satiety,-10);AddValue(Spirit,-9)`、`Priority` 高於歸零 Game Over 事件的列即可（**每個階段開始**都扣，含開局與 SetPhase 跳轉）。參考 SampleData 事件 188。

**LocationData（地點）**：`ID`、`Name`/`Description`、`VisibleCondition`（出現在移動選單的條件）、`ShowInMenu`（0=不出現在選單）、`SortOrder`、`Note`

**PlayerActionData（玩家行動）**：`ID`（`AfterAction:{ID}` 引用）、`Name`/`Description`、`Locations`（可出現地點 ID 逗號分隔，空白=所有地點）、`VisibleCondition`、`EnableCondition`（不符合時按鈕反灰）、`Commands`、`SortOrder`、`Note`。慣例：純機制行動效果直接寫 Commands；劇情行動 Commands 留空、由 `AfterAction:{ID}` 事件依條件決定結果。

**GameEventTriggerData（事件觸發，劇情流程核心）**：`ID`（`$EventDone_{ID}` 引用）、`Timing`、`Condition`、`Priority`（同時機多事件命中時大→小依序執行）、`OneTime`（1=一生一次，自動記 `EventDone_{ID}=1`）、`DialogueID`（0=無對話）、`PrePerformance`/`PostPerformance`（演出 ID，空白=無）、`Commands`、`Note`。
執行順序：**PrePerformance → 對話 → Commands → PostPerformance**；每個事件執行前都會**重新檢查條件**（前面的事件可能改變數值）。
⚠️ 互斥條件事件（如次數 1/2/3+ 分別觸發）請把「較晚期」的條件設較高 Priority，避免前一個事件把數值改到符合下一個條件而連續觸發。

**GameTextData（遊戲文字）**：`ID`（`ShowHint(ID)` 引用）、`Group`（`Monologue(群組)` 抽選用）、`Condition`、`Text`（可用 `\n`）、`Weight`（抽選權重，預設 1）、`Note`

**DialogueData（劇情對話，DialogueSystem 格式）**：`ID`（同段劇情共用）、`Line`（從 1 開始）、`Command`、`Arg1`~`Arg5`、`Arg1_en` 等多語欄位（可留空）。常用指令：`Say(說話者,內容)`（說話者空=旁白）、`AddOption(選項文字,目標對話ID,目標行號)`、`ShowOptions()`、`GoToLine(行號)`、`BlackIn/BlackOut(秒)`、`ShowFullScreenImage(圖名,淡入秒)`（Addressables 鍵值）、`GameEffect(效果指令串)`（**GameFlowSystem 擴充**：對話中執行遊戲效果指令）。完整內建清單見 `DialogueSystem/README.md`。

## 3. View 層（9 個腳本）

所有全螢幕/彈窗 View 繼承 `KahaGameCore.UserInterfaceSystem.AView`（提供 `Show(CancellationToken)` / `Hide(CancellationToken)` 與 CanvasGroup 淡入淡出；`transitionDuration` 序列化欄位控制秒數）。Item 類（按鈕項目）是普通 MonoBehaviour。

以下程式碼全文可直接抄（namespace 換成自己的）。

### 3.1 MainMenuView（主標題）

```csharp
using System;
using KahaGameCore.UserInterfaceSystem;
using TMPro;
using UnityEngine;
using UnityEngine.UI;

namespace YourGame.Presentation.Views
{
    /// <summary>主標題畫面。僅負責顯示與轉發按鈕事件，不含流程邏輯。</summary>
    public class MainMenuView : AView
    {
        public event Action OnStartRequested;

        [SerializeField] private TextMeshProUGUI titleText;
        [SerializeField] private Button startButton;

        private void Awake()
        {
            startButton.onClick.AddListener(() => OnStartRequested?.Invoke());
        }

        public void SetTitle(string title)
        {
            titleText.text = title;
        }
    }
}
```

### 3.2 GameplayHudView + StatValueItem（HUD）

```csharp
using System;
using System.Collections.Generic;
using System.Threading;
using Cysharp.Threading.Tasks;
using KahaGameCore.UserInterfaceSystem;
using TMPro;
using UnityEngine;

namespace YourGame.Presentation.Views
{
    /// <summary>遊戲主 HUD：天數/時段、數值狀態列、自言自語浮動文字。</summary>
    public class GameplayHudView : AView
    {
        [SerializeField] private TextMeshProUGUI dayPhaseText;
        [SerializeField] private RectTransform statContainer;
        [SerializeField] private StatValueItem statItemPrefab;
        [SerializeField] private CanvasGroup monologueGroup;
        [SerializeField] private TextMeshProUGUI monologueText;
        [SerializeField] private float monologueFadeDuration = 0.3f;
        [SerializeField] private float monologueHoldDuration = 2.5f;

        private readonly Dictionary<string, StatValueItem> tagToStatItem = new Dictionary<string, StatValueItem>();
        private CancellationTokenSource monologueCts;

        public void SetDayPhase(string text)
        {
            dayPhaseText.text = text;
        }

        public void BindStats(IReadOnlyList<(string tag, string displayName, int value)> stats)
        {
            foreach (StatValueItem item in tagToStatItem.Values)
            {
                Destroy(item.gameObject);
            }
            tagToStatItem.Clear();

            foreach ((string tag, string displayName, int value) in stats)
            {
                StatValueItem item = Instantiate(statItemPrefab, statContainer);
                item.Bind(tag, displayName, value);
                tagToStatItem.Add(tag, item);
            }
        }

        public bool TryUpdateStat(string tag, int value)
        {
            if (!tagToStatItem.TryGetValue(tag, out StatValueItem item))
            {
                return false;
            }
            item.UpdateValue(value);
            return true;
        }

        public void ShowMonologue(string text)
        {
            monologueCts?.Cancel();
            monologueCts?.Dispose();
            monologueCts = new CancellationTokenSource();
            PlayMonologueAsync(text, monologueCts.Token).Forget();
        }

        private async UniTaskVoid PlayMonologueAsync(string text, CancellationToken token)
        {
            monologueText.text = text;
            monologueGroup.gameObject.SetActive(true);

            await FadeMonologueAsync(0f, 1f, token);
            await UniTask.Delay(TimeSpan.FromSeconds(monologueHoldDuration), cancellationToken: token);
            await FadeMonologueAsync(1f, 0f, token);

            monologueGroup.gameObject.SetActive(false);
        }

        private async UniTask FadeMonologueAsync(float from, float to, CancellationToken token)
        {
            float timer = 0f;
            while (timer < monologueFadeDuration)
            {
                token.ThrowIfCancellationRequested();
                timer += Time.deltaTime;
                monologueGroup.alpha = Mathf.Lerp(from, to, timer / monologueFadeDuration);
                await UniTask.Yield(token);
            }
            monologueGroup.alpha = to;
        }

        private void OnDestroy()
        {
            monologueCts?.Cancel();
            monologueCts?.Dispose();
        }
    }
}
```

### 3.3 ActionMenuView + ActionButtonItem（行動選單）

`ActionMenuEntry` / `IGameFlowAction` 來自包核心（`KahaGameCore.Package.GameFlowSystem`）。

```csharp
using System;
using System.Collections.Generic;
using KahaGameCore.Package.GameFlowSystem;
using KahaGameCore.UserInterfaceSystem;
using UnityEngine;

namespace YourGame.Presentation.Views
{
    /// <summary>行動選單：依表格資料動態產生行動按鈕。</summary>
    public class ActionMenuView : AView
    {
        [SerializeField] private RectTransform buttonContainer;
        [SerializeField] private ActionButtonItem buttonPrefab;

        private readonly List<ActionButtonItem> spawnedButtons = new List<ActionButtonItem>();

        public void Bind(IReadOnlyList<ActionMenuEntry> entries, Action<ActionMenuEntry> onSelected)
        {
            ClearButtons();
            foreach (ActionMenuEntry entry in entries)
            {
                ActionButtonItem button = Instantiate(buttonPrefab, buttonContainer);
                button.Bind(entry, onSelected);
                spawnedButtons.Add(button);
            }
        }

        private void ClearButtons()
        {
            foreach (ActionButtonItem button in spawnedButtons)
            {
                Destroy(button.gameObject);
            }
            spawnedButtons.Clear();
        }
    }
}
```

```csharp
using System;
using KahaGameCore.Package.GameFlowSystem;
using TMPro;
using UnityEngine;
using UnityEngine.UI;

namespace YourGame.Presentation.Views
{
    /// <summary>行動選單上的單一行動按鈕。</summary>
    public class ActionButtonItem : MonoBehaviour
    {
        [SerializeField] private Button button;
        [SerializeField] private TextMeshProUGUI nameText;
        [SerializeField] private TextMeshProUGUI descriptionText;

        public void Bind(ActionMenuEntry entry, Action<ActionMenuEntry> onClicked)
        {
            nameText.text = entry.Action.Name;
            descriptionText.text = entry.Action.Description;
            button.interactable = entry.IsEnabled;
            button.onClick.RemoveAllListeners();
            button.onClick.AddListener(() => onClicked?.Invoke(entry));
        }
    }
}
```

### 3.4 LocationMenuView + LocationButtonItem（移動選單）

`LocationData` 來自 `KahaGameCore.Package.GameFlowSystem.DefaultImplements.Data`。

```csharp
using System;
using System.Collections.Generic;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.Data;
using KahaGameCore.UserInterfaceSystem;
using UnityEngine;
using UnityEngine.UI;

namespace YourGame.Presentation.Views
{
    /// <summary>移動選單：列出可前往的地點，附取消（返回）按鈕。</summary>
    public class LocationMenuView : AView
    {
        [SerializeField] private RectTransform buttonContainer;
        [SerializeField] private LocationButtonItem buttonPrefab;
        [SerializeField] private Button cancelButton;

        private readonly List<LocationButtonItem> spawnedButtons = new List<LocationButtonItem>();
        private Action onCancelled;

        private void Awake()
        {
            cancelButton.onClick.AddListener(() => onCancelled?.Invoke());
        }

        public void Bind(IReadOnlyList<LocationData> locations, Action<LocationData> onSelected, Action onCancelled)
        {
            this.onCancelled = onCancelled;
            ClearButtons();
            foreach (LocationData location in locations)
            {
                LocationButtonItem button = Instantiate(buttonPrefab, buttonContainer);
                button.Bind(location, onSelected);
                spawnedButtons.Add(button);
            }
        }

        private void ClearButtons()
        {
            foreach (LocationButtonItem button in spawnedButtons)
            {
                Destroy(button.gameObject);
            }
            spawnedButtons.Clear();
        }
    }
}
```

`LocationButtonItem` 與 `ActionButtonItem` 同構（`Bind(LocationData, Action<LocationData>)`，設定 `location.Name` / `location.Description`，點擊轉發），照抄改型別即可。

## 4. Presenter 層

Presenter 是「包介面 ↔ View」的轉接層，全部是純 C#（非 MonoBehaviour）。共通模式：**以 `UniTaskCompletionSource` 等待玩家操作；`CancelPending()` 讓返回標題時等待中的選擇以 null/完成 收場，流程才能退出**。

實作的包介面：`IActionMenuPresenter`（核心）、`IHintPresenter`、`ILocationMenuPresenter`（DefaultImplements）。

### 4.1 ActionMenuPresenter

```csharp
using System;
using System.Collections.Generic;
using System.Threading;
using Cysharp.Threading.Tasks;
using KahaGameCore.Package.GameFlowSystem;
using YourGame.Presentation.Views;

namespace YourGame.Presentation.Presenters
{
    public class ActionMenuPresenter : IActionMenuPresenter
    {
        private readonly ActionMenuView view;
        private UniTaskCompletionSource<IGameFlowAction> pendingSelection;

        public ActionMenuPresenter(ActionMenuView view)
        {
            this.view = view ? view : throw new ArgumentNullException(nameof(view));
        }

        public async UniTask<IGameFlowAction> SelectActionAsync(IReadOnlyList<ActionMenuEntry> entries)
        {
            CancelPending();
            pendingSelection = new UniTaskCompletionSource<IGameFlowAction>();

            view.Bind(entries, entry => pendingSelection.TrySetResult(entry.Action));
            await view.Show(CancellationToken.None);

            IGameFlowAction selected = await pendingSelection.Task;

            await view.Hide(CancellationToken.None);
            return selected;
        }

        /// <summary>結束遊戲流程時呼叫，讓等待中的選擇以 null 結束以利流程退出。</summary>
        public void CancelPending()
        {
            pendingSelection?.TrySetResult(null);
        }
    }
}
```

### 4.4 GameplayHudPresenter（HUD 更新）

訂閱 EventBus 事件更新 HUD；`Dispose()` 時退訂。

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using KahaGameCore.GameData.Implemented;
using KahaGameCore.GameEvent;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.Data;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.Events;
using YourGame.Presentation.Views;

namespace YourGame.Presentation.Presenters
{
    /// <summary>監聽遊戲狀態事件並更新 HUD。顯示哪些數值由 GameValueData.ShowInHUD 決定。</summary>
    public class GameplayHudPresenter : IDisposable
    {
        private readonly GameplayHudView view;
        private readonly IGameState gameState;
        private readonly ITimeService timeService;
        private readonly List<GameValueData> hudValueDefinitions;

        public GameplayHudPresenter(GameplayHudView view, GameStaticDataManager staticDataManager,
            IGameState gameState, ITimeService timeService)
        {
            this.view = view ? view : throw new ArgumentNullException(nameof(view));
            this.gameState = gameState ?? throw new ArgumentNullException(nameof(gameState));
            this.timeService = timeService ?? throw new ArgumentNullException(nameof(timeService));

            GameValueData[] definitions = staticDataManager.GetAllGameData<GameValueData>();
            hudValueDefinitions = definitions == null
                ? new List<GameValueData>()
                : definitions.Where(d => d.ShowInHUD == 1).OrderBy(d => d.ID).ToList();

            EventBus.Subscribe<GameValueChangedEvent>(OnGameValueChanged);
            EventBus.Subscribe<TimePhaseChangedEvent>(OnTimePhaseChanged);
            EventBus.Subscribe<MonologueRequestedEvent>(OnMonologueRequested);
        }

        /// <summary>開新遊戲時重建狀態列。</summary>
        public void Refresh()
        {
            view.BindStats(hudValueDefinitions
                .Select(d => (d.Tag, d.DisplayName, gameState.Get(d.Tag)))
                .ToList());
            UpdateDayPhaseText();
        }

        public void Dispose()
        {
            EventBus.Unsubscribe<GameValueChangedEvent>(OnGameValueChanged);
            EventBus.Unsubscribe<TimePhaseChangedEvent>(OnTimePhaseChanged);
            EventBus.Unsubscribe<MonologueRequestedEvent>(OnMonologueRequested);
        }

        private void OnGameValueChanged(GameValueChangedEvent e) => view.TryUpdateStat(e.Tag, e.NewValue);
        private void OnTimePhaseChanged(TimePhaseChangedEvent e) => UpdateDayPhaseText();
        private void OnMonologueRequested(MonologueRequestedEvent e) => view.ShowMonologue(e.Text);

        private void UpdateDayPhaseText()
        {
            if (timeService.CurrentPhase == null) return;
            view.SetDayPhase($"第 {timeService.CurrentDay} 天　{timeService.CurrentPhase.DisplayName}");
        }
    }
}
```

## 5. 組裝根 GameLauncher

唯一的 MonoBehaviour 入口：載表 → 建 Presenter → `GameFlowSystemBuilder` 組裝 → 控制「主標題 ↔ 遊戲流程」切換。

```csharp
using System.Threading;
using Cysharp.Threading.Tasks;
using KahaGameCore.GameData.Implemented;
using KahaGameCore.GameEvent;
using KahaGameCore.Package.GameFlowSystem;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.Data;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.DataAccess;
using KahaGameCore.Package.GameFlowSystem.DefaultImplements.Events;
using KahaGameCore.UserInterfaceSystem;
using ProjectBSR.DialogueSystem;
using ProjectBSR.DialogueSystem.View;
using UnityEngine;
using YourGame.Presentation.Presenters;
using YourGame.Presentation.Views;

namespace YourGame
{
    /// <summary>組裝根（Composition Root）：所有相依關係只在這裡建立。</summary>
    public class GameLauncher : MonoBehaviour
    {
        private const string MAIN_MENU_VIEW_PATH = "UIViews/MainMenuView";
        private const string GAMEPLAY_HUD_VIEW_PATH = "UIViews/GameplayHudView";
        private const string ACTION_MENU_VIEW_PATH = "UIViews/ActionMenuView";
        private const string LOCATION_MENU_VIEW_PATH = "UIViews/LocationMenuView";
        private const string HINT_POPUP_VIEW_PATH = "UIViews/HintPopupView";
        private const string CREDITS_VIEW_PATH = "UIViews/CreditsView";

        [SerializeField] private UserInterfaceController uiController;
        [SerializeField] private DialogueView dialogueView;
        [Tooltip("行動選單、提示視窗等覆蓋層 View 的父節點。")]
        [SerializeField] private RectTransform overlayRoot;
        [Tooltip("七張表的 JSON TextAsset（檔名需與型別名一致）。改用 Resources 載入時可留空。")]
        [SerializeField] private TextAsset[] gameDataTables;
        [SerializeField] private string gameTitle = "My Game";
        [Tooltip("製作人員名單文字（GameTextData 表的 ID）。")]
        [SerializeField] private int creditsTextId = 950;

        private GameStaticDataManager staticDataManager;
        private GameFlowServices services;
        private ActionMenuPresenter actionMenuPresenter;
        private LocationMenuPresenter locationMenuPresenter;
        private HintPresenter hintPresenter;
        private GameplayHudPresenter hudPresenter;
        private CancellationTokenSource flowCts;
        private bool isGameRunning;

        private void Awake()
        {
            // 視窗失焦時仍持續運作（轉場與演出皆以時間驅動，暫停會卡住流程）。
            Application.runInBackground = true;

            LoadStaticData();
            EventBus.Subscribe<ReturnToTitleRequestedEvent>(OnReturnToTitleRequested);
        }

        private void Start()
        {
            ShowMainMenuAsync().Forget();
        }

        private void OnDestroy()
        {
            EventBus.Unsubscribe<ReturnToTitleRequestedEvent>(OnReturnToTitleRequested);
            CancelFlow();
            hudPresenter?.Dispose();
        }

        private void LoadStaticData()
        {
            staticDataManager = new GameStaticDataManager();
            // 手動指定版；改用 Resources 時換成 new ResourcesJsonStaticDataHandler()
            TextAssetJsonStaticDataHandler handler = new TextAssetJsonStaticDataHandler(gameDataTables);
            GameFlowSystemBuilder.LoadDefaultTables(staticDataManager, handler);
            staticDataManager.Add<DialogueData>(handler);
        }

        private async UniTaskVoid ShowMainMenuAsync()
        {
            await uiController.ClearViewStack();
            MainMenuView mainMenu = await uiController.PushView<MainMenuView>(
                MAIN_MENU_VIEW_PATH,
                view => view.SetTitle(gameTitle));
            mainMenu.OnStartRequested += () => StartGameAsync().Forget();
        }

        private async UniTaskVoid StartGameAsync()
        {
            if (isGameRunning) return;
            isGameRunning = true;

            EnsureServicesBuilt();

            await uiController.ClearViewStack();
            GameplayHudView hudView = await uiController.PushView<GameplayHudView>(GAMEPLAY_HUD_VIEW_PATH);

            hudPresenter?.Dispose();
            hudPresenter = new GameplayHudPresenter(hudView, staticDataManager, services.GameState, services.TimeService);
            hudPresenter.Refresh();

            flowCts = new CancellationTokenSource();
            services.FlowController.RunNewGameAsync(flowCts.Token).Forget();
        }

        private void EnsureServicesBuilt()
        {
            if (services != null) return;

            actionMenuPresenter = new ActionMenuPresenter(InstantiateOverlayView<ActionMenuView>(ACTION_MENU_VIEW_PATH));
            locationMenuPresenter = new LocationMenuPresenter(InstantiateOverlayView<LocationMenuView>(LOCATION_MENU_VIEW_PATH));
            hintPresenter = new HintPresenter(InstantiateOverlayView<HintPopupView>(HINT_POPUP_VIEW_PATH));

            // 全部採用預設實作；有專案特殊需求時改用 Override 系列方法傳入。
            services = new GameFlowSystemBuilder(staticDataManager)
                .WithDialogueView(dialogueView)
                .WithActionMenuPresenter(actionMenuPresenter)
                .WithHintPresenter(hintPresenter)
                .WithLocationMenuPresenter(locationMenuPresenter)
                .Build();

            RegisterPerformances();
        }

        /// <summary>演出註冊處：表格引用的演出 ID 在這裡註冊；未註冊的 ID 以佔位 Log 代替，不卡流程。</summary>
        private void RegisterPerformances()
        {
            CreditsView creditsView = InstantiateOverlayView<CreditsView>(CREDITS_VIEW_PATH);
            services.PerformancePlayer.Register("Credits", new CreditsPerformance(creditsView, services.TextProvider, creditsTextId));
        }

        private T InstantiateOverlayView<T>(string resourcePath) where T : AView
        {
            T prefab = Resources.Load<T>(resourcePath);
            if (prefab == null)
            {
                Debug.LogError($"[GameLauncher] 找不到 View prefab：Resources/{resourcePath}");
                return null;
            }
            T view = Instantiate(prefab, overlayRoot);
            view.gameObject.SetActive(false);
            return view;
        }

        private void OnReturnToTitleRequested(ReturnToTitleRequestedEvent requestedEvent)
        {
            CancelFlow();

            // 兩件事缺一不可：取消 token 停掉流程迴圈與事件佇列；
            // CancelPending 讓停在選單上的 await 以 null 收場。
            actionMenuPresenter.CancelPending();
            locationMenuPresenter.CancelPending();
            hintPresenter.CancelPending();

            hudPresenter?.Dispose();
            hudPresenter = null;

            dialogueView.gameObject.SetActive(false);
            isGameRunning = false;

            ShowMainMenuAsync().Forget();
        }

        private void CancelFlow()
        {
            if (flowCts != null)
            {
                flowCts.Cancel();
                flowCts.Dispose();
                flowCts = null;
            }
        }
    }
}
```

注意：主選單/HUD 走 `UserInterfaceController.PushView(Resources路徑)`（View 堆疊管理），覆蓋層（行動/移動選單、提示、名單）走 `InstantiateOverlayView` 常駐實例。View prefab 因此**必須放在 Resources 下**（上方常數路徑）；只有表格資料可以選擇手動指定。

## 8. 啟動驗收與疑難排解

按 Play 應依序看到：主標題 → 點「開始遊戲」 → GameStart 事件（開場對話）→ HUD（天數/時段/數值）→ 行動選單。

| 症狀 | 原因 / 解法 |
|---|---|
| 中文全是方塊 | TMP 無 CJK 字型 → 自建中文 Font Asset |
| 按鈕點不動 | EventSystem 缺 `InputSystemUIInputModule`，或 Active Input Handling 沒設 Both（設完要重啟） |
| 對話點不下去 | 同上（DialogueView 用舊版 Input）；或編輯器失焦——`Application.runInBackground = true` 已寫在 GameLauncher.Awake |
| `找不到資料檔` / `未指定 ○○ 的表格 TextAsset` | 表格檔名必須與型別名一致；手動指定模式檢查 Inspector 陣列 |
| `沒有任何可選行動，自動推進時間` 警告 | PlayerActionData 的 Locations/VisibleCondition 沒涵蓋目前地點/時段 |
| 點開始後畫面全黑 | View prefab 不在 Resources 路徑（看 Console 的 `[GameLauncher] 找不到 View prefab`） |
| 事件沒觸發 | Timing 字串拼錯（`PhaseStart:{Key}` 的 Key 要跟 TimePhaseData 一致）；OneTime 事件已記 `EventDone_{ID}` |
| 指令參數含文字壞掉 | EffectProcessor 會剝除半形空格——文字放 GameTextData 以 ID 引用 |

## 9. 之後的客製化

- 換掉任一預設服務：實作對應介面 → `GameFlowSystemBuilder.OverrideXxx(custom)`（見 README「快速接入」）。
- 加效果指令：實作 `EffectCommandBase` → `AddCommandRegistration(c => c.RegisterFactory("名稱", new DelegateEffectCommandFactory(() => new 你的指令(...))))`。
- 對話中做遊戲效果：DialogueData 用 `GameEffect(指令串)`（DefaultImplements 已註冊）。
- 流程骨架本身不符需求（如不要時段制）：實作核心 8 介面自組，見 README 後半。
